# MTSIS Lite Frontend

Легкий фронт для главной логики проекта:
- турист видит маршруты (линии на карте) без перегруза POI
- по умолчанию карта пустая, маршрут показывается только после выбора
- выбирает маршрут и нажимает "Начать маршрут"
- при старте обязательно указывается телефон и expected return
- видит свои check-ins и может сделать checkout
- ranger/admin видят мониторинг активных check-ins
- H3-сетку можно включать/выключать (используется resolution 9)
- авторизация в демо работает без Bearer токена (через `X-Demo-Role`)


Открыть в браузере:

## 4) Что тестировать

### Роль `hiker`
1. Выбрать роль `hiker` в панели
2. Выбрать маршрут слева
3. Нажать `Начать маршрут`
4. Проверить блок `Мои check-ins`
5. Нажать `Завершить маршрут (checkout)`

### Роль `ranger`
1. Выбрать роль `ranger`
2. Включить `Check-ins`
3. Проверить таблицу мониторинга (panel `Мониторинг`)
4. Нажать `Отметить overdue`

### Роль `admin`
1. Выбрать роль `admin`
2. Повторить мониторинг check-ins
3. Дополнительно проверить `/docs`

## 5) Переключатели карты

- `H3 сетка`: OFF по умолчанию (включается при необходимости)
- `POI`: OFF по умолчанию
- `Check-ins`: OFF по умолчанию

Это сделано специально, чтобы турист видел читаемые маршруты без визуального шума.

## 6) Мониторинг для ranger/admin

- В таблице мониторинга видны:
  - check-in id
  - user id
  - название маршрута + сложность + длина
  - телефон
  - expected return
  - статус
- Клик по строке мониторинга сразу открывает соответствующий маршрут на карте.
